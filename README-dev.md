# README-dev.md

## понятия

- `!!s587s-base!!` - база данных Airtable по имени "дневник s411s"
- `!!s587s-tr-table!!` - таблица в `!s587s-base!` по имени "доход расход"
    - `!!s587s-trs-field!!` - поле "счёт списания линк" таблицы `!s587s-tr-table!`; значения в этом поле имеют тип Link на таблицу `!s587s-acc-table!`
    - `!!s587s-trn-field!!` - поле "счёт назначения линк" таблицы `!s587s-tr-table!`; значения в этом поле имеют тип Link на таблицу `!s587s-acc-table!`
    - `!!s587s-trd-field!!` - поле "дата" таблицы `!s587s-tr-table!`; тип поля - `Date` (Airtable)
    - `!!s587s-trssum-field!!` - поле "сумма списания" таблицы `!s587s-tr-table!`; тип поля - `Number` (Airtable)
    - `!!s587s-trzsum-field!!` - поле "сумма назначения" таблицы `!s587s-tr-table!`; тип поля - `Number` (Airtable)
    - `!!s587s-trcom-field!!` - поле "комментарий" таблицы `!s587s-tr-table!`; тип поля - `Long text` (Airtable)
- `!!s587s-acc-table!!` - таблица в `!s587s-base!` по имени "доход расход категории"
    - `!!s587s-accname-field!!` - поле `name` таблицы `!s587s-acc-table!`; в этом поле хранятся названия счетов которые мы используем в таблице `!s587s-tr-table!` в полях `!s587s-trs-field!` и `!s587s-trs-field!`
- `!!s587s-ui-main-page!!` - главная страница
    - `!s587s-ui-as-field!` - поле "Счёт списания"; выпадающий список с фильтрацией; пользователь начинает вводить название и появляются подсказки значений из `!s587s-accname-field!`; под полем всегда отображаются последние три значения которые пользователь выбирал 
    - `!s587s-ui-an-field!` - поле "Счёт зачисления"; работает также как поле `!s587s-ui-as-field!`

### `!!s587s-ui-tr-page!!` - страница просмотра всех транзакций и управления ими

- основой текущей страницы является таблица `!s587s-ui-tr-page!`
- на текущей странице доступнен полный CRUD всех полей
- доступна пагинация

#### `!!s587s-ui-f-block!!` - блок фильтраций

- `!!s587s-ui-f-field!!` - поле быстрой фильтрации по комментариям; введённый текст нужно сопоставлять с полем `!s587s-trcom-field!` 
- `!!s587s-ui-df-block!!` - блок из полей для фильтрации по дате; состоит из поля ввода даты `!!s587s-ui-ds-field!!` и поля ввода даты `!!s587s-ui-de-field!!`
  - логика работы
    - если оба поля не заполнены - не фильтриуем
    - если одно из полей пустое, а другое нет, то фильтруем по полному совпадению заполненного поля с `!s587s-trd-field!`
    - если оба поля заполнены то фильтруем по диапазону дат: выбираем записи, где значение поля `!s587s-trd-field!` находится в интервале между `!s587s-ui-ds-field!` включительно и `!s587s-ui-de-field!` включительно

### блок отображения транзакций

- содержит список блоков `!s587s-ui-trt-block!`
- у списка есть пагинация

#### `!!s587s-ui-trt-block!!` - UI отражающий отдельную транзакцию

- отображает значения полей:
  - `!s587s-trs-field!`
  - `!s587s-trn-field!`
  - `!s587s-trd-field!`
  - `!s587s-trssum-field!`
  - `!s587s-trzsum-field!`
  - `!s587s-trcom-field!`
